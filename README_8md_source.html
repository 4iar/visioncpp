4 # Overview
5 **VisionCpp** is a lightweight header-only library for computer vision and image processing.
6 The aim of the library is to provide a toolbox that enables performance portability for heterogeneous platforms using modern C++.
7 
8 Written using [SYCL 1.2](https://www.khronos.org/registry/sycl/specs/sycl-1.2.pdf) and compiled/tested with [ComputeCpp](https://codeplay.com/products/computesuite/computecpp) to accelerate vision code using OpenCL devices.

29 ## <a name="sample-code" /> Sample Code
30 Below is a very simple application that will do the conversion RGB -> HSV. Full source code can be found in the examples folder.
31 RGB is assumed to be a three-channel unsigned char storage with a reasonable channel order.
32 
33 ~~~~~~~~~~~~~~~{.cpp}
34  // main, args, checks and all the boring stuff
35 
36  // ...
37 
38  // where VisionCpp will run.
39  auto dev = visioncpp::make_device<visioncpp::backend::sycl,
40  visioncpp::device::cpu>();
41 
42  // create a host container for input data
43  std::shared_ptr<unsigned char> in_rgb(new unsigned char[3],
44  [](unsigned char *dataMem) { delete[] dataMem;});
45 
46  in_rgb.get()[0] = atoi(argv[1]);
47  in_rgb.get()[1] = atoi(argv[2]);
48  in_rgb.get()[2] = atoi(argv[3]);
49 
50  // create a host container for output data
51  std::shared_ptr<unsigned char> out_hsv(new unsigned char[3],
52  [](unsigned char *dataMem) { delete[] dataMem;});
53 
54  // exiting this scope will sync data
55  {
56  // definition of the VisionCpp pipeline:
57 
58  // create terminal nodes - a leaf node ( data node ) of the expression tree.
59  // terminal struct takes 4 arguments
60  // 1st template parameter specifies the data U8 (unsigned char) C3 (three
61  // channels)
62  // 2nd number of columns in the storage
63  // 3rd number of rows in the storage
64  // 4th underlying storage type - currently only Buffer2D supported
65  auto data =
66  visioncpp::terminal<visioncpp::pixel::U8C3, 1, 1,
67  visioncpp::memory_type::Buffer2D>(in_rgb.get());
68  auto data_out =
69  visioncpp::terminal<visioncpp::pixel::U8C3, 1, 1,
70  visioncpp::memory_type::Buffer2D>(out_hsv.get());
71 
72  // unsigned char -> float RGB storage conversion
73  auto node = visioncpp::point_operation<visioncpp::OP_U8C3ToF32C3>(data);
74  // float RGB to float HSV conversion
75  auto node2 = visioncpp::point_operation<visioncpp::OP_RGBToHSV>(node);
76  // helper node that allows display of HSV
77  // for unsigned char: V <- 255*V, S <- 255*S, H <- H/2 ( to fit in range of 0..255 )
78  auto node3 = visioncpp::point_operation<visioncpp::OP_HSVToU8C3>(node2);
79 
80  // assign operation that writes output of the pipe to output terminal node
81  auto pipe = visioncpp::assign(data_out, node3);
82  // execute the pipeline
83  // 1st template parameter defines if VisionCpp back-end fuses the expression
84  // 2nd & 3rd shared memory sizes ( column, row )
85  // 4th & 5th local work group size ( column , row )
86  visioncpp::execute<visioncpp::policy::Fuse, 1, 1, 1, 1>(pipe, dev);
87  }
88 
89  printf("RGB: %u %u %u \nHSV: %u %u %u \n", in_rgb.get()[0], in_rgb.get()[1],
90  in_rgb.get()[2], out_hsv.get()[0], out_hsv.get()[1], out_hsv.get()[2]);
91 
92 ~~~~~~~~~~~~~~~

94 ## <a name="requirements" /> Requirements
95 To successfully compile VisionCpp tests, you will need:
96 * ComputeCpp (https://codeplay.com/products/computesuite/computecpp)
97 * OpenCV 3.2 (https://github.com/opencv/opencv) - used for camera access, window display and as a testing reference.
98 * GTest (https://github.com/google/googletest) - testing framework.
99 * OpenCL 1.2

101 ## <a name="build" /> Build
102 Assuming you are in the root of a git repo:
103 ~~~~~~~~~~~~~~~{.sh}
104 mkdir build
105 cd build
106 cmake .. -DCOMPUTECPP_PACKAGE_ROOT_DIR={PATH_TO_COMPUTECPP_ROOT} -DCMAKE_CXX_COMPILER={FAVORITE_CXX_COMPILER}
107 make -j8
108 make test
109 ~~~~~~~~~~~~~~~
110 
111 The output binaries will be catalogued in bin folder.
112 ~~~~~~~~~~~~~~~{.sh}
113 | - build
114  | - bin
115  | - example
116  | - test
117 ~~~~~~~~~~~~~~~
